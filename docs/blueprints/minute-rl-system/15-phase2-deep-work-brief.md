# 15 第二阶段深度优化任务简报（00:00-02:00）

时区：`Asia/Shanghai`

## 1. 阶段目标

1. 在不破坏 `P0-FREEZE` 的前提下，进一步补齐可执行细节与验收路径。
2. 输出“可以直接进入实现排期”的文档包，而非抽象建议。
3. 对所有关键争议给出明确裁决逻辑、阈值与回滚策略。

## 2. 硬约束（不可破）

1. 不改主因果契约与三时钟一致性阈值。
2. 不改已冻结幂等键与唯一真相源规则。
3. 不把 paper 证据误写为 live 可行性证据。
4. 不引入超大新体系导致明日执行链路返工。

## 3. 必答问题（来自阶段一红蓝质询）

1. 在线 OOD 检测与冻结规则如何定义（指标、阈值、连续窗口）。
2. reward 在部分成交/延迟回填下的唯一归因算法如何落表。
3. 降级与恢复如何避免振荡（冷却时间、二次触发策略）。
4. 模拟器校准结果如何绑定适用域并管理有效期。
5. 晋级裁决如何处理“收益好但执行略劣”的灰区。

## 4. 预期输出

1. 章节级补丁清单（按文件列出修改建议）。
2. 指标定义表（字段、计算口径、阈值、触发动作）。
3. 验收清单（可执行、可判定 pass/fail）。
4. 需要人工签字的决策点列表（与自动裁决边界）。
