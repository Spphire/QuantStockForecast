# 01 目标与边界

## 1. 核心目标

1. 在不破坏当前日频主线（`research -> white-box -> submit -> briefing`）的前提下，新增分钟级 paper 子系统。
2. 复用既有范式：`expert -> ensemble/voting -> white-box risk -> execution`。
3. 在分钟级链路中引入 RL 在线学习，但先做“元决策控制”，不直接逐笔控制订单。
4. 形成可灰度、可回滚、可审计的上线机制。

## 2. 明确不做

1. 不改写现有日频调度脚本与任务。
2. 不在首版让 RL 直接决定逐股订单价格/数量。
3. 不在没有稳定 A/B 与执行质量证据前切真实账户。

## 3. 上线原则

1. 先稳后快：先运行安全基线（`Bandit + white-box`），再推进 `IQL + SAC`。
2. 先可审计再可优化：任何收益改进必须能被审计字段回放验证。
3. 先防幻觉再追收益：时间因果、奖励归因、模拟器校准不通过则不允许晋级。
