# 25 配置漂移与变更控制 Runbook

## 场景 E1：配置摘要不一致

### 触发

1. 核心进程 `config_digest` 不一致。
2. 缺少 `approved_change_id` 或审批元信息。
3. 运行中核心参数发生漂移。

### 自动处置

1. 立即冻结 challenger。
2. 切 `dry-run only`。
3. 当日污染样本不入训练与晋级。

### 恢复

1. 配置回滚到审批版本。
2. `config_digest` 全进程一致。
3. 变更审批与证据补齐后，先 shadow 再恢复。

## 场景 E2：手工操作越权

### 触发

1. 非白名单账户执行关键操作。
2. 未经审批脚本改写运行参数。
3. 手工覆盖门禁导致流程跳步。

### 自动处置

1. 阻断后续自动执行。
2. 触发审计告警并记录 `override_reason`。
3. 强制进入人工复核。

### 恢复

1. 完成越权根因复盘。
2. 恢复白名单与权限边界。
3. 完成回归演练后再恢复自动链路。
