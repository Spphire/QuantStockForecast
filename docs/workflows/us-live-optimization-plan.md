# 美股实战优化计划（仅 US，暂停数据集 Ablation）

更新时间：2026-03-23（Asia/Shanghai）

## 1. 目标与边界

### 1.1 核心目标（明日实战优先）
- 以美股实盘链路稳定性为第一优先：`research -> submit -> execution -> 开盘执行简报`。
- 在不破坏现有可运行流程的前提下，提升策略可比性、可解释性与上线安全性。
- 参考同事结果中的有效方向（`regressor + rank/mean ensemble`），但仅采用仓库内可控、可回滚实现。

### 1.2 明确不做
- 暂停“数据集 ablation（A股 vs A+美）”。
- 暂停大规模重构与重训全矩阵。
- 暂停非美股链路优先级工作。

### 1.3 成功标准（明日上线口径）
- 全流程可一键跑通，任意环节失败都有降级路径。
- 简报稳定产出（含异常说明），不因局部失败导致“静默”。
- 评估口径统一：同一时间窗、同一 SPY benchmark、同一会话掩码。

## 2. 并行分工（Subagent 任务）

> 主线程负责调度、集成、验收与最终决策。

### SA-A（数据与特征）
- 任务：修复/规避“模型间特征缺失处理不一致”导致的训练样本口径漂移。
- 重点：
  - 树模型与序列模型的缺失值策略一致化。
  - 明确 `turnover/cs_rank_turnover` 在美股源缺失时的处理规则。
  - 确保不会因 `dropna(feature_columns)` 把有效美股样本整批筛空。
- 交付：
  - 统一口径说明文档。
  - 训练样本统计（rows/symbols/date range）对齐校验脚本。

### SA-B（模型与集成）
- 任务：将当前 `vote` 集成升级为“保留排序信息”的实战可用版本。
- 重点：
  - 引入/强化 `rank_average` 与 `mean_score` 的主路径。
  - `vote` 仅作对照，不作为默认上线策略。
  - 提供可回滚的策略开关（配置级切换）。
- 交付：
  - 默认集成方案 + 备选方案（各自回滚方式）。
  - 对照结果表（Sharpe、回撤、换手、成本）。

### SA-C（执行与风控）
- 任务：保障“开盘执行”环节鲁棒性。
- 重点：
  - 订单失败（休市、价格越界、部分成交、超时）处理一致。
  - 提交后仓位应通过 Alpaca 实际回查，而非理论仓位。
  - 关键保护：`session guard`、`kill switch`、重复执行防护。
- 交付：
  - P0 阻塞清单。
  - 降级策略清单（自动继续/中止并告警）。

### SA-D（评估协议与上线门禁）
- 任务：给出明日上线前“最小且严格”的门禁。
- 重点：
  - Walk-forward 最小可行验证。
  - SPY 统一基准与统一比较区间。
  - 上线前 hard-gates 与失败降级路径。
- 交付：
  - Go/No-Go 清单。
  - 上线后监控阈值与触发动作。

## 3. 主线程 60 秒心跳机制

### 3.1 心跳节奏
- 每 60 秒汇总一次各 SA 状态：`running / blocked / done`。
- 每次心跳输出 4 项：
  - 已完成项
  - 新发现风险
  - 阻塞点
  - 下一分钟动作

### 3.2 心跳升级规则
- 连续 2 次无产出：主线程下发“80/20 快速交付”指令。
- 连续 3 次无产出：中断深挖任务，切换为“先交可执行结论，细节后补”。
- 若仍无响应：回收任务并由主线程接管该子任务。

## 4. 优化执行路线（按优先级）

### P0（必须今天完成）
1. 统一训练/评估口径
- 统一 benchmark：真实 `SPY buy&hold`。
- 统一会话掩码：全部策略同一比较窗口。
- 统一样本统计：每次训练输出 rows/symbol/date 范围。

2. 集成策略切换
- 默认从 `vote` 切到 `rank_average`（保留排序信息）。
- 保留 `mean_score` 作为第二候选。
- `vote` 仅保留为对照，不用于默认上线。

3. 执行简报稳定性
- 订单失败/超时仍发简报（显式“执行状态与残缺项”）。
- 仓位分布以“成交后 Alpaca 实际持仓”为准。
- 账户权益曲线统一为“现金+持仓市值”时间序列。

### P1（今日可做，若时间不足可延后）
1. 专家加权集成
- 以最近 walk-forward 表现作为动态权重。
- 对明显弱势专家设置权重下限或熔断开关。

2. 特征健壮化
- 对美股缺失字段采用统一 neutral fill + 显式 missing indicator。
- 补齐序列模型与树模型特征处理一致性。

3. 监控与告警细化
- 增加成交率、订单拒绝率、简报完整率、数据新鲜度告警。

### P2（实战后迭代）
- 更复杂 stacking / meta-learner。
- 多市场联合训练的系统性 domain adaptation。
- 更细粒度交易成本与滑点建模。

## 5. 上线前门禁（Go/No-Go）

### 5.1 必须满足
- 全流程 dry-run 通过（research + submit + briefing）。
- Benchmark 对齐检查通过（SPY、窗口、会话数一致）。
- 关键文件产出完整（预测、风控、执行、简报）。
- 异常路径演练通过（超时/部分成交/订单拒绝仍可收敛）。

### 5.2 触发 No-Go 的条件
- 比较口径不一致（任一策略会话或 benchmark 不一致）。
- 执行后无法获得真实仓位快照。
- 简报在失败场景下无法产出。
- 关键保护（session guard / kill switch）失效。

## 6. 实战当日运行顺序（简版）
1. 预检查：数据新鲜度、市场状态、账户状态、风险开关。
2. 运行 research（US-only）。
3. 运行 submit 与成交回补。
4. 生成并发送开盘执行简报（含异常注记）。
5. 开盘后监控与阈值告警。

## 7. 回滚与降级策略
- 模型降级：`rank_average -> mean_score -> 单模型 lightgbm`。
- 执行降级：自动缩小仓位 -> 暂停新单 -> 仅监控。
- 简报降级：若图表失败，发送纯文本关键指标 + 异常详情，保证不静默。
