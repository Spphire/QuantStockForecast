# Cleaning

`data_module/cleaning` 是预留的数据清洗模块，目前还没有独立脚本落地，但它在整体架构里很重要。

## 计划职责

- 缺失值处理
- 异常值裁剪
- 复权口径统一
- 多数据源字段对齐
- 停牌、涨跌停、极端成交量等规则化清洗
- 训练前的样本有效性过滤

## 为什么现在还没独立出来

当前项目还在快速验证期，很多清洗逻辑直接写在：

- 数据抓取脚本
- LightGBM 特征工程脚本
- 白盒风控过滤逻辑

这种方式能更快跑通实验，但随着模块增多，会让职责边界变得模糊。

## 未来推荐拆分方式

- `price_cleaning.py`
  行情字段清洗
- `liquidity_cleaning.py`
  流动性相关过滤
- `calendar_alignment.py`
  多市场、多股票时间对齐
- `label_safety.py`
  避免未来信息泄露的样本检查

## 当前状态

- 规划中
- 尚未形成独立可调用接口

## 何时应该优先实现

当你开始出现下面这些需求时，应该优先补这个模块：

- 不同市场混合训练
- 多源数据拼接
- 新闻、财报、行情联合建模
- 更严格的 walk-forward 研究
